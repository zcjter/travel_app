# 旅游纪念册平台 (个人版) 需求文档

## 1. 项目概述
*   **项目名称**：云途 (CloudJourney) - 个人版
*   **目标**：构建一个私有化的、以地理信息驱动的深度旅行记录平台。
*   **技术栈**：
    *   **前端**：Vue 3 (Composition API) + Pinia + Vite + Element Plus + Leaflet.js (地图)。
    *   **后端**：FastAPI + SQLAlchemy + Pydantic。
    *   **存储**：SQLite/PostgreSQL + 本地文件系统/Minio。

---

## 2. 核心功能需求

### 2.1 旅程管理模块 (Trip Management)
作为记录的最高层级，管理不同时间、不同地点的旅行项目。
*   **【增】创建旅程**：支持设置旅程名称、起止日期、封面图、预算、目的地城市。
*   **【查】旅程概览**：以时间线或网格卡片形式展示所有旅程，支持按年份筛选。
*   **【改】旅程编辑**：修改基础信息、更换封面、调整旅程状态（进行中/已完成/计划中）。
*   **【删】级联删除**：删除旅程时，可选是否同时物理删除该旅程下的所有媒体文件。

### 2.2 瞬间/日志记录模块 (Moments/Daily Log)
在每一段旅程内，按时间点或地点进行的碎片化记录。
*   **【增】发布记录**：支持文字随笔、地理打卡、多图上传。
*   **【查】沉浸式阅读**：支持“时间轴模式”和“地图模式”切换。
*   **【改】记录修改**：支持补记文字、重新调整地理坐标。
*   **【删】单条删除**：删除特定的瞬间记录。

### 2.3 媒体库与自动化 (Media & Automation)
*   **智能上传**：用户上传照片后，后端利用 `exifread` 库**自动解析**：
    *   拍摄时间（自动填充记录时间）。
    *   GPS 坐标（自动在地图上标点）。
    *   设备型号（记录相机/手机型号、光圈、快门等参数）。
*   **自动天气回溯**：根据照片中的“坐标+时间”，后端异步调用历史天气接口（如 OpenWeather），自动补齐当时的**天气、温度、风力**信息。
*   **媒体查看**：集成 Vue-viewer，支持高清大图缩放、旋转及视频播放。

### 2.4 足迹可视化模块 (Footprint Visualization)
*   **交互式足迹地图**：
    *   **全局模式**：在首页展示所有去过的点位和连线。
    *   **旅程模式**：进入单次旅程后，地图自动缩放至该行程区域，展示每日移动轨迹。
*   **轨迹自动连线**：系统根据 Moments 的时间顺序，自动计算逻辑路径并生成平滑连线。

### 2.5 个人数据看板 (Dashboard)
*   **多维统计**：
    *   **足迹统计**：去过的省份、城市数量。
    *   **里程统计**：旅程跨度总距离估计。
    *   **媒体统计**：存储的照片总数、视频总时长。
    *   **时间分布**：年度旅行频率分布柱状图。

## 3. 数据库设计 (Schema)

### 3.1 Trips (旅程表)
| 字段        | 类型    | 说明          |
| :---------- | :------ | :------------ |
| id          | Integer | 主键          |
| title       | String  | 旅程名称      |
| cover_image | String  | 封面图URL     |
| start_date  | Date    | 开始时间      |
| end_date    | Date    | 结束时间      |
| description | Text    | 旅程寄语/概括 |

### 3.2 Moments (记录表)
| 字段          | 类型     | 说明                 |
| :------------ | :------- | :------------------- |
| id            | Integer  | 主键                 |
| trip_id       | Integer  | 关联旅程ID (外键)    |
| content       | Text     | 文字内容             |
| location_name | String   | 地点名称             |
| lat / lon     | Float    | 经纬度               |
| weather_info  | JSON     | 存储天气、温度等数据 |
| created_at    | DateTime | 记录产生的时间       |

### 3.3 Media (媒体表)
| 字段      | 类型    | 说明              |
| :-------- | :------ | :---------------- |
| id        | Integer | 主键              |
| moment_id | Integer | 关联记录ID (外键) |
| file_path | String  | 文件存储路径      |
| file_type | String  | image/video       |
| exif      | JSON    | 存储相机参数信息  |

## 4. API 接口设计 (FastAPI)

| 动作       | 路径             | 功能说明                               |
| :--------- | :--------------- | :------------------------------------- |
| **GET**    | `/trips`         | 获取所有旅程列表                       |
| **POST**   | `/trips`         | 创建新旅程                             |
| **GET**    | `/trips/{id}`    | 获取旅程详情（含所有Moments）          |
| **POST**   | `/upload`        | 核心接口：上传媒体，返回EXIF与预估位置 |
| **POST**   | `/moments`       | 提交一条记录（关联图片与文字）         |
| **GET**    | `/stats/summary` | 获取仪表盘统计数据                     |
| **DELETE** | `/moments/{id}`  | 删除单条记录                           |

## 5. 前端页面结构 (Vue 3)

1.  **首页 (Home)**：
    *   全屏地图展示历史足迹。
    *   下方瀑布流展示“最近的旅行”。
2.  **旅程管理页 (Trip List)**：
    *   卡片式列表，支持按年份和关键词搜索。
3.  **详情页 (Trip Detail)**：
    *   **左侧**：垂直时间轴，显示文字和照片。
    *   **右侧**：该旅程的局部地图，显示路线。
4.  **发布页 (Editor)**：
    *   支持图片拖拽上传。
    *   自动从图片提取位置并显示在地图微缩预览中。
    *   文本域记录感悟。

---

## 6. 非功能性需求

*   **性能优化**：
    *   后端必须生成**缩略图**（Thumbnail）。前端在列表页加载缩略图，详情页加载原图，防止内存溢出。
    *   使用 `Lazy-loading` 处理长途旅行的大量图片。
*   **私密性与安全**：
    *   前端集成简单的登录验证（JWT）。
    *   后端支持环境变量配置访问密钥，防止接口被恶意调用。
*   **可移植性**：
    *   支持 Docker 部署。
    *   提供 SQLite 数据库备份与媒体文件一键导出的脚本。

### 开发优先级建议：
1.  **第一步**：搭建 FastAPI 基础框架，完成 `Trips` 和 `Moments` 的基础 CRUD 接口。
2.  **第二步**：实现文件上传逻辑，并集成 EXIF 读取功能（这是提升个人体验的关键）。
3.  **第三步**：编写 Vue 前端，先实现文字和图片的展示。
4.  **第四步**：集成地图库 (Leaflet/Mapbox)，将数据可视化。
5.  **第五步**：增加统计看板。
